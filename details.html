Design Details

Scope Priority Ladder

Everything on Day 2 is structured around one principle: the FSDP2 smoke test never slips. If we run out of time, we slip from the bottom of this ladder, not the top. The smoke test answers the single biggest architectural risk in V1 — failing to get a verdict on it is worse than shipping a partial training demo.

Priority Item Required for the smoke test? Required for the training-E2E demo? Slips when behind?
1 Package installable (pip install -e .) + ForgeRMSNorm ported Yes Yes Never
2 Smoke test script (tests/test_fsdp2_smoke.py) staged and runnable Yes No Never
3 Patching infra (forge.patch/forge.unpatch) working No Yes If H10 is past 13:30
4 Qwen3 patches: RMSNorm + SwiGLU + Embedding No Yes RoPE drops first if H10 slips
5 Gemma patches: RMSNorm(offset=1.0) + GeGLU + Embedding No Yes Demo with Qwen3 only if H11 slips
6 Qwen3 LoRA SFT training E2E (500 steps, loss decreases) No Yes Reduce to 200 steps before dropping
7 Gemma LoRA SFT training E2E No Yes Drop before Qwen3
8 Integration tests (single-step + convergence) both models No Receipts Single-step only, skip convergence
9 (bottom) RoPE patch + LoRA-fused patches + CE/Fused-Linear-CE patch No Stretch Drop first
The Rule
  • Items 1 + 2 MUST be done by 13:00 Sunday. P3 (smoke test runner) cannot start without them. Period.
  • Items 3 – 5 are the demo. If we ship these, we have a working "two architectures, kernels active" story even without convergence numbers.
  • Item 9 is acceptable to drop. CE/RoPE/LoRA-fused patches are Phase 2 work in the V1 plan anyway — pulling them in is a bonus, not a target.
  • Never trade smoke-test readiness for more kernel patches. A passing smoke test + Qwen3-only demo > full kernel patches + no smoke test.

Locked Decisions on Layout
  • No dynamic registry. We have 9 kernels, not 50 — explicit imports are not painful. The patching mapping files (qwen3.py, gemma.py) ARE the de-facto registry. Skipping the registry removes one layer of indirection from the codebase.
  • Package directory at ./forge/, not ./src/forge/. Simpler. No src-layout learning curve. Liger-Kernel uses the same flat layout.
  • One kernel file = one Triton kernel + one autograd.Function + one nn.Module. Do NOT split across files. Reading rmsnorm.py top-to-bottom should give the full picture.
  • SwiGLU and GeGLU live in the SAME file (kernels/swiglu.py). The activation function is a tl.constexpr branch — compile-time, zero runtime overhead. P1 builds the SiLU path, P6 builds the GELU path, both PR into this one file.

Patching Architecture

How forge.patch(model) actually works. There are two valid approaches — we chose forward replacement (monkey-patching). The other approach (module replacement) is explicitly rejected.

Approach What it does Pro Con Decision
Module replacement Walk model.named_modules(), replace Qwen2RMSNorm instances with ForgeRMSNorm instances (copy the weight) Clean, type-correct Breaks HF state_dict loading. Confuses FSDP2's wrapping policies (which match on the original class name). Rejected
Forward replacement Walk modules, monkey-patch module.forward = forge_forward_closure(module). Preserves the original module object. State_dict compatible. FSDP2-friendly. Reversible via unpatch. Already validated in integration_gpt2large/integration_test.ipynb. Monkey-patching is "ugly" but bounded. Chosen

Concrete pattern

# forge/patching/core.py
def patch(model, kernels=None):
    """Patch HF model in place. Returns the same model object.

    Args:
        model: HF causal LM (Qwen3 or Gemma).
        kernels: Optional list of kernel names to enable.
                 None = enable all available. Useful for debugging:
                 forge.patch(model, kernels=["rmsnorm"]) enables ONE kernel.
    """
    if hasattr(model, '_forge_patched'):
        raise RuntimeError("Model already patched. Call unpatch first.")

    arch = _detect_architecture(model)        # "qwen3" or "gemma"
    mapping = _load_mapping(arch)              # imports qwen3.py or gemma.py

    model._forge_originals = {}                  # module_id -> original_forward
    for name, module in model.named_modules():
        cls_name = type(module).__name__
        if cls_name in mapping:
            kernel_name, config = mapping[cls_name]
            if kernels is not None and kernel_name not in kernels:
                continue
            _replace_forward(module, kernel_name, config, model._forge_originals)

    model._forge_patched = True
    return model


def unpatch(model):
    """Restore original forwards. Idempotent."""
    if not hasattr(model, '_forge_patched'):
        return model
    for module_id, original_fwd in model._forge_originals.items():
        module = _id_to_module(model, module_id)
        module.forward = original_fwd
    del model._forge_originals
    del model._forge_patched
    return model


# THE CLOSURE FACTORY — critical to avoid Python's late-binding loop bug
def _replace_forward(module, kernel_name, config, originals):
    originals[id(module)] = module.forward
    forge_module = _build_forge_kernel(kernel_name, config, module)
    module.forward = _make_forward(forge_module)

def _make_forward(forge_module):
    def forward(*args, **kwargs):
        return forge_module(*args, **kwargs)
    return forward                          # closure binds forge_module correctly


def _detect_architecture(model):
    model_type = model.config.model_type     # set by HF
    if model_type == "qwen2": return "qwen3"   # Qwen2.5 reports "qwen2"
    if model_type == "gemma2": return "gemma"
    raise ValueError(f"Unsupported architecture: {model_type}")
Locked Decisions on Patching
  • Forward replacement (monkey-patch module.forward), NOT module replacement. Already validated in the GPT-2 integration test.
  • Idempotency: Double-patch raises RuntimeError. We do NOT silently no-op. Failing loud catches the bug where someone forgets to unpatch in a test fixture.
  • Selective patching: forge.patch(model, kernels=["rmsnorm"]) is supported from Day 2 onward. This is the #1 debugging tool when training diverges — enable kernels one at a time to bisect the broken one.
  • Closure factory pattern is mandatory. Do NOT write module.forward = lambda x: forge_module(x) directly inside a loop — Python's late binding will silently break it. Use _make_forward().

Exact mapping from HF Qwen2/Qwen3 module class names to Forge kernels. P8 owns this file on Day 2.

# forge/patching/qwen3.py

QWEN3_MAPPING = {
    # MUST-SHIP (priority items 4 in the scope ladder)
    "Qwen2RMSNorm":    ("rmsnorm",     {"offset": 0.0}),
    "Qwen2MLP":         ("swiglu_mlp",  {"activation": "silu"}),    # patches whole MLP forward
    "Embedding":        ("embedding",   {}),                          # nn.Embedding on embed_tokens

    # STRETCH (priority 9 — drop if behind)
    # "Qwen2Attention":   ("attention_with_forge_rope", {"base": 10000.0}),
    # CE / Fused-Linear-CE are loss-step interceptions, not module patches.
    # Deferred to Phase 2 of the V1 plan.
}

Why these three are the must-ship choices

Kernel Patch target Difficulty Day 2 status
RMSNorm Qwen2RMSNorm.forward Easy — same as GPT-2 LayerNorm pattern Must
SwiGLU MLP Qwen2MLP.forward (replace whole MLP) Easy — Qwen2MLP.__init__ exposes gate_proj/up_proj/down_proj as attributes; we call them inside our replacement. Must
Embedding nn.Embedding.forward of embed_tokens Easy — single tensor lookup Must
RoPE Qwen2Attention.forward (have to rewrite parts of it) Medium — RoPE is called INSIDE attention forward, must surgically replace just the rotary call Stretch
Cross-Entropy Hook the loss step (when labels are passed) Hard — requires intercepting model.forward output handling Defer
Fused Linear+CE Same as CE plus the lm_head Linear Hard — coupled loss + projection step Defer
LoRA MLP/QKV fused Replace PEFT's Linear.forward AFTER LoRA wrap Hard — couples with PEFT internals Defer

Patching the whole MLP (not just the activation) is intentional — it matches how Liger and Unsloth structure their patches. The MLP replacement code knows about gate_proj, up_proj, down_proj as module attributes and calls them inside the Triton-backed forward. This is cleaner than trying to splice into Qwen's F.silu(gate) * up call site.

Training Loop (Custom, ~40 lines)

We use a custom training loop, NOT HF Trainer. Decision rationale is in the table below. P1 owns examples/sft_qwen3_lora.py and examples/sft_gemma_lora.py on Day 2.

Approach Pros Cons Decision
HF Trainer 5 lines of config. Handles mixed precision, scheduler, logging, gradient accumulation. Same code path as V1 ships. Opaque when things break. Silently skips NaN steps. Errors come from 6+ frames deep in accelerate. Forces Dataset/DataCollator conventions. Rejected
Custom loop Total transparency. NaN explodes immediately. Trivial to bisect kernel bugs by toggling kernels=. ~40 lines, top-to-bottom readable. More boilerplate. Distributed-ready requires extra code (not needed for hackathon). Chosen

Why custom for the hackathon: we WILL hit kernel correctness bugs during training. A black-box trainer makes those bugs harder to diagnose. The ~50 lines of training script are throwaway — the kernels (the valuable artifact) translate fully to V1's HF Trainer + FSDP2 path.

The script (Qwen3 version — Gemma is a one-line model-id change)

# examples/sft_qwen3_lora.py
import torch, forge
from torch.utils.data import DataLoader
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import get_peft_model, LoraConfig
from datasets import load_dataset

torch.manual_seed(42)

# 1. Load model (small for hackathon iteration speed)
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-0.5B", torch_dtype=torch.bfloat16,
).cuda()
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-0.5B")
tokenizer.pad_token = tokenizer.eos_token

# 2. forge.patch BEFORE LoRA — this ordering is critical
forge.patch(model)                                # enable all available kernels
# For debugging, narrow scope: forge.patch(model, kernels=["rmsnorm"])

# 3. LoRA wrap (after patch — PEFT's adapter wrappers go OUTSIDE our patched forwards)
peft_config = LoraConfig(
    r=16, lora_alpha=32,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],
    task_type="CAUSAL_LM",
)
model = get_peft_model(model, peft_config)

# 4. Tiny dataset slice (smollm-corpus, pre-cached Friday night)
ds = load_dataset("HuggingFaceTB/smollm-corpus", "cosmopedia-v2", split="train[:2000]")
def tok(ex):
    out = tokenizer(ex["text"], truncation=True, max_length=512, padding="max_length")
    out["labels"] = out["input_ids"].copy()
    return out
ds = ds.map(tok, batched=True, remove_columns=ds.column_names)
ds.set_format("torch")
loader = DataLoader(ds, batch_size=4, shuffle=True)

# 5. The loop — 500 steps
optim = torch.optim.AdamW([p for p in model.parameters() if p.requires_grad], lr=2e-4)
model.train()
losses = []
for step, batch in enumerate(loader):
    if step >= 500: break
    batch = {k: v.cuda() for k, v in batch.items()}
    out = model(**batch)
    out.loss.backward()
    optim.step()
    optim.zero_grad()
    losses.append(out.loss.item())
    if step % 10 == 0:
        print(f"step {step}: loss {out.loss.item():.4f}")
    assert not torch.isnan(out.loss), f"NaN at step {step}"   # fail loud, fail fast

# 6. Success criterion
assert losses[-1] < losses[0] * 0.9, f"Loss didn't decrease 10%: {losses[0]:.3f} -> {losses[-1]:.3f}"
print("✅ Demo passed")
Done When (per model)
  • Script runs for 500 steps without NaN, without crash
  • Final loss < 90% of initial loss
  • Wall-clock under 10 minutes on a single H100
  • Script with forge.patch(model) removed (vanilla baseline) also runs successfully — this confirms our changes vs the baseline
Locked Decisions on Training
  • Custom loop, not HF Trainer. Debuggability over convenience for the hackathon.
  • Dataset: HuggingFaceTB/smollm-corpus (cosmopedia-v2 split, first 2000 rows). Pre-download in Friday checklist. Synthetic random tokens rejected — real text catches more bugs (padding, attention masks, etc).
  • 500 steps, not 200. Worth the extra ~7 minutes for a credible loss curve.
  • Critical ordering: forge.patch(model) MUST happen BEFORE get_peft_model. PEFT wraps base modules in adapter layers — if we patch after, our patched forwards never fire.
  • Filter optimizer params: [p for p in model.parameters() if p.requires_grad]. PEFT marks LoRA params trainable and base params frozen — we want AdamW only on LoRA.
  • Fail loud: assert not torch.isnan(loss) at every step. Unlike Trainer, we do NOT silently skip NaN steps.

Integration Tests

Two tests per model. P2 owns these on Day 2. These are the receipts that forge.patch doesn't silently corrupt the model.

Test 1 — Single-step correctness

# tests/test_integration_qwen3.py
def test_patched_vs_unpatched_logits():
    """Patched model should produce numerically equivalent logits to unpatched."""
    m_ref   = load_qwen3()
    m_forge = load_qwen3(); forge.patch(m_forge)

    torch.manual_seed(42)
    x = torch.randint(0, 151936, (1, 128), device='cuda')
    out_ref   = m_ref(x).logits
    out_forge = m_forge(x).logits

    max_diff   = (out_ref - out_forge).abs().max().item()
    cos_sim    = F.cosine_similarity(out_ref.flatten(), out_forge.flatten(), dim=0).item()
    top1_match = (out_ref.argmax(-1) == out_forge.argmax(-1)).float().mean().item()

    assert max_diff < 1.0,    f"Max logit diff {max_diff} too high"
    assert cos_sim > 0.999,   f"Cosine sim {cos_sim} too low"
    assert top1_match > 0.95, f"Top-1 agreement {top1_match} too low"

Test 2 — Convergence (100 steps, optional)

# tests/test_convergence_qwen3.py  (drop to single-step if behind schedule)
def test_loss_curves_match():
    """Patched and unpatched training produce similar loss curves over 100 steps."""
    losses_ref   = train_loop(forge_patch=False, steps=100, seed=42)
    losses_forge = train_loop(forge_patch=True,  steps=100, seed=42)

    for i, (l_ref, l_forge) in enumerate(zip(losses_ref, losses_forge)):
        diff_pct = abs(l_ref - l_forge) / l_ref
        assert diff_pct < 0.10, f"Step {i}: loss diverged by {diff_pct:.1%}"
Tolerance Starting value Why this number What to do if it fails
max_diff < 1.0 (logits) 1.0 Logits are unbounded; raw absolute diff is noisy in bf16. 1.0 is loose but catches catastrophic kernel bugs. Bisect kernels with forge.patch(model, kernels=[one]). Tighten threshold as you isolate.
cos_sim > 0.999 0.999 Direction-of-logit-vector matters more than magnitude for next-token prediction. If 0.99 < sim < 0.999: kernel has minor numerical drift, acceptable. If < 0.99: real bug.
top1_match > 0.95 0.95 The end-user-visible metric. 95% agreement means the model picks the same top token 95% of positions. If < 0.95: a kernel is changing the argmax. Bisect.
diff_pct < 10% (loss) 10% per step Loss curves accumulate divergence; 10% per step is forgiving but flags drift. If > 10% by step 20: gradient bug in some kernel.

These tolerances are starting points. Real numbers from running the test will tell us where to tighten. The bar in rules.txt is "no measurable accuracy loss" — we'll align tolerances with that bar based on what we actually observe.
